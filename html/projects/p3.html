PhotoApp

Overview:

A web service application built using Node.js and the Express framework, designed to act as an intermediary between a Python-based client and AWS services. The service handles requests from the client, manages interactions with S3 for image storage, and RDS (MySQL) for user and image tracking, while enforcing security through IAM policies. The web service optimizes the workflow by decoupling the client from direct AWS interaction, improving scalability and flexibility.

Technologies:

  • Frontend: Python-based client
  • Backend: Node.js with Express framework
  • Database: AWS RDS (MySQL)
  • Storage: AWS S3 for image storage
  • Deployment: AWS Elastic Beanstalk and EC2 for web service hosting
  • Security: AWS IAM for managing access permissions

Features:

  • Introduced a web service tier between the Python client and AWS services.
  • Enabled API-based image upload, retrieval, and user management.
  • Stored images in AWS S3, and managed user and image data using RDS (MySQL).
  • API endpoints for fetching statistics, user details, assets, and downloading images.

Role:

  • Developed the Node.js web service to handle client requests and manage AWS services.
  • Integrated the Python client to communicate with the new web service rather than AWS directly.
  • Deployed the web service using AWS Elastic Beanstalk, ensuring scalability and accessibility.
  • Configured security roles using AWS IAM for managing permissions between the web service and AWS services.

Challenges:

  • To ensure smooth communication between the Python client and web service, I implemented and tested API routes to handle client requests and interact with AWS services.
  • To handle image uploads and downloads securely and efficiently, I used AWS S3 for storage, incorporating efficient retrieval and encoding mechanisms like Base64 for secure data transfer.

Takeaways

  • Gained experience in building multi-tier, cloud-native applications with AWS services.
  • Learned the importance of API design in separating the client from backend services for better scalability.
  • Understood the benefits of using AWS Elastic Beanstalk for simplifying deployment and management of cloud-based applications.
