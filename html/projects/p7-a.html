Project A

Overview:

Create dynamic and interactive 3D toy models using WebGL. The goal was to design and implement two distinct jointed assemblies: a bike and a helicopter, each with animated parts and user-interactive controls. The focus was on building complex 3D parts, connecting them with sequential joints, and providing smooth animations and interactivity for the user. The project captures the essence of classic toys in a digital environment, giving users a nostalgic, playful experience with modern WebGL technologies.

Technologies:

  • WebGL: Used to render the 3D models and animations in the browser.
  • JavaScript: For managing the logic behind the jointed assemblies, transformations, and user interactions.
  • HTML5 Canvas: To create the space where the WebGL scenes and interactive elements are displayed.
  • Matrix Transformations (cuon-matrix-quat.js or glMatrix.js): Libraries were used to handle 3D transformations for the parts and assemblies.
  • Shader Programming: Vertex and fragment shaders were written for managing per-vertex colors and lighting effects.

Features:

  • Two Distinct 3D Assemblies: The bike and helicopter are two independently moving, jointed 3D assemblies.
    • The bike features two sequential joints (handlebars and seat post).
    • The helicopter includes rotors and blades that move based on user input.
  • Interactive Controls:
    • Users can rotate the helicopter by clicking and dragging.
    • The helicopter blades can be rotated with the left and right arrow keys, and the rotors with the up and down arrow keys.
    • Smooth animations for both assemblies, ensuring no jerky transitions between movements.

Demo:

Role:

  • Designing the Parts: Created custom 3D parts for both the bike and the helicopter, ensuring they met the complexity requirements (12+ vertices) and enclosed a 3D volume.
  • Matrix Transformations: Utilized matrix transformations to control the positioning, scaling, and rotations of the parts, ensuring proper animation and alignment of hinged joints.
  • Interactivity and Animation: Implemented event handlers to allow user interaction with the helicopter's controls and managed the smooth, automated animations for the bike.

Challenges:

  • Challenge: Managing Sequential Joints
    • Solution: The most complex part was handling sequential joints, especially for the bike where the seat and handlebars needed to rotate independently yet still be connected. This was solved by using a hierarchical transformation matrix stack, ensuring that each part could move independently without affecting the rest of the assembly.
  • Challenge: Implementing Smooth User Interaction
    • Solution: Event handlers were used for mouse and keyboard inputs, but ensuring smooth control of the helicopter's rotation required careful handling of input rates and the smoothness of the rotation. JavaScript’s requestAnimationFrame was used to guarantee fluid transitions.
  • Challenge: Ensuring Consistent Frame Rates for Smooth Animation
    • Solution: Optimized the WebGL code to handle all rendering within the GPU, reducing the number of calls made per frame and improving performance by storing vertices in Vertex Buffer Objects (VBOs), allowing them to be transferred to the GPU only once.

Takeaways

  • Understanding of WebGL’s Matrix Stack: Gained a deep understanding of how to use WebGL’s matrix stack to manage hierarchical transformations in a jointed assembly. The project emphasized the importance of correctly pushing and popping matrices when dealing with complex 3D models with multiple joints.
  • Interactive Animation Techniques: This project honed skills in creating user-interactive animations with WebGL, especially focusing on how to link real-time inputs (mouse/keyboard) to visual changes in 3D space
  • User Experience Design: Emphasized the importance of clear instructions and intuitive controls to ensure that users can easily interact with 3D models, an essential takeaway for future interactive WebGL projects.
