Formula

Overview:

Developed software for monitoring and analyzing various aspects of a vehicle’s performance through a series of interconnected sensor boards under the Data Acquisition System (DAQ) subgroup. The DAQ system normalized and transmitted sensor data via the Controller Area Network (CAN) bus, allowing other systems to track vehicle conditions like temperature, wheel speed, and brake pressure. The project included modular boards for each key system and a logger board for data storage and analysis. The primary goal was to gather reliable sensor data for real-time performance monitoring and analysis.

Technologies:

  • Languages: C++
  • Hardware: ESP32 microcontroller, sensor boards (temperature, wheel speed, brake pressure, etc.)
  • PlatformIO
  • TSMaster

Features:

  • Modular System: The DAQ system consisted of multiple daughter boards (e.g., wheel speed, brake pressure, temperature) designed for specific functions. Each board measured and transmitted data to CAN
  • Each board collected sensor data, applied dimensionalization algorithms, and sent the processed data onto the High Priority (HP) or Low Priority (LP) CAN bus.
  • Data Logging: The Logger Board read data from the CAN bus and stored it on an SD card for later analysis.
  • Flexible Design: The system was designed for scalability, with over 10 daughter boards, allowing for simpler electrical harness manufacturing and reliable data readings.
  • Advanced Sensor Interpretation: Non-linear sensor data was processed using interpolation techniques such as quadratic or cubic spline fits.

Role:

  • Developed software for multiple sensor boards, including those for wheel speed, brake pressure, and temperature. This involved writing dimensionalization algorithms, configuring sensors, and handling CAN bus data transmission.
  • Worked closely with the hardware team to understand how the physical boards and sensors operated. This collaboration helped ensure that the software was aligned with hardware functionality and that all components worked seamlessly together.
  • The CAN library to manage data transmission between the sensor boards and the CAN bus, guaranteeing that the system functioned correctly across different modules.

Challenges:

  • Learning the CAN Library: One major challenge was learning how to use the CAN library effectively for transmitting and receiving data across various boards. To overcome this, I spent time understanding the protocol and performing tests to ensure data integrity and compatibility across the system.
  • Understanding Sensor Boards: To ensure accurate data collection, I had to thoroughly test and understand each sensor’s behavior, from wheel speed to brake pressure. This required careful experimentation, building custom test rigs, and iterating on the dimensionalization algorithms.
  • Hardware Coordination: Coordination with the hardware team was critical to ensure that the software could operate with the physical sensor boards. I had regular check-ins with the hardware team to discuss prototype availability and clarify expected sensor behaviors.
  • Documentation and Code Structure: Given the complexity of the system and the cross-team collaboration, maintaining clean code and thorough documentation was essential. This ensured other team members could easily follow the logic and modify the system as needed.

Takeaways

  • Cross-Team Communication: Effective communication between hardware and software teams was essential for ensuring project success. I learned how important it is to synchronize efforts between both sides and manage expectations effectively.
  • CAN Bus Proficiency: This project allowed me to become proficient in working with the CAN protocol, which is crucial for real-time data transmission in vehicle systems.
  • Modular Design Thinking: Designing modular, scalable systems enabled more straightforward troubleshooting and future enhancements. The modular structure also simplified electrical harness manufacturing and improved reliability.
  • Code Quality and Maintainability: Ensuring that code is easy to understand and well-documented became a critical skill during this project, especially as the system complexity increased with multiple daughter boards.
