Project B

Overview:

The project focuses on creating a large, interactive 3D world that users can explore using a movable 3D camera. The user can control the camera's position and orientation to navigate the world, which includes multiple jointed assemblies. The canvas displays two side-by-side views—one using a perspective projection and the other using an orthographic projection—highlighting the differences between these two views. The scene includes a grid-like ground plane and animated, sequentially-jointed 3D parts. The project also introduces interactive elements such as sliders and checkboxes for controlling joint angles and objects via keyboard and mouse inputs.

Technologies:

  • WebGL: Used for rendering the 3D scene, creating objects, and managing projections.
  • JavaScript: For handling user input, event listeners, camera controls, and scene updates.
  • HTML5 Canvas: For displaying the interactive 3D world in the browser.
  • Matrix Transformations (glMatrix.js): Utilized for managing object transformations and camera movements.
  • Shader Programming: Custom shaders for handling the lighting, projections, and vertex transformations in the scene.

Features:

  • Interactive Camera: Users can move the camera in all directions using WASD keys to strafe and the arrow keys to pan, creating intuitive navigation.
  • Dual Viewports: The canvas is split into two views:
    • Perspective View: Shows the 3D world using a perspective projection.
    • Orthographic View: Displays the same 3D world but with an orthographic projection, allowing users to compare the two views side by side.
  • Ground Plane: A large, grid-like ground plane extends across the 3D space, helping users orient themselves as they move the camera.
  • Jointed Assemblies: Includes multiple animated assemblies with at least four sequential joints, which can be controlled using sliders and checkboxes.
  • Mouse Interaction: Users can click and drag within the canvas to rotate a stationary object, mimicking trackball-like control over its orientation.
  • Manual Joint Control: Sliders and checkboxes allow users to manually control the joint angles of the 3D assemblies.

Demo:

Role:

  • 3D World Creation: Designed and built the 3D world, including the ground plane and several animated jointed assemblies.
  • Camera System Development: Implemented an interactive camera system that allowed users to explore the scene using keyboard and mouse inputs. Ensured that the camera movement was intuitive, allowing for both aiming and movement in the world.
  • Projection Handling: Set up the dual viewports with one using a perspective projection and the other an orthographic projection. Developed the logic to ensure both views showed the scene consistently.
  • Event Handling: Integrated controls for moving the camera, adjusting joint angles, and interacting with the scene through mouse and keyboard inputs.

Challenges:

  • Dual Viewports: Ensuring that both the perspective and orthographic views rendered correctly with the same camera parameters was challenging. I solved this by carefully adjusting the near and far clipping planes and ensuring the correct projection matrices were applied to each view.
  • Camera Movement: Creating a camera that could both aim in any direction and move through the 3D space without changing its orientation was challenging. This was addressed by separating the camera’s aiming controls from its movement, allowing for independent control of direction and position.
  • User Interaction: Implementing smooth controls for rotating objects via mouse input, and adjusting joint angles with sliders, required careful integration of event listeners. Solving this involved using WebGL's matrix stack for smooth transitions and responsive object manipulation.

Takeaways

  • Perspective vs. Orthographic Understanding: Gained a deeper understanding of how perspective and orthographic projections influence 3D rendering and user perception of space. This project emphasized the practical differences between these views and how to implement them efficiently in WebGL.
  • Camera and Navigation Techniques: Improved my ability to design intuitive camera controls that allow for free movement and aiming in 3D space. Understanding how to decouple movement from aiming direction made the camera system far more flexible.
  • User Interaction Design: Learned how to integrate user controls such as mouse-drag rotations, keyboard navigation, and GUI sliders into a WebGL project. This enhanced the interactivity and user experience of the application.
  • Matrix Transformations: Strengthened my skills in working with transformation matrices, particularly in managing sequential joint rotations and camera positioning within a dynamic 3D scene.
